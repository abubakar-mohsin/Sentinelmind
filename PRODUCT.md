# SentinelMind

## Register
product

## Product Purpose
Real-time AI cybersecurity platform dashboard. Monitors live security events, runs a multi-agent AI pipeline analyzing threats, displays incidents, threat intelligence, and automated response actions. University project demonstrating enterprise security architecture.

## Users
Security engineers and senior technical people. They live in terminals and tools like Datadog, Linear, and Vercel. Zero tolerance for bloat, decoration, or anything that looks like a template. They care about data density, speed, and clarity.

## Brand Voice
Precise, technical, understated. No marketing language. No exclamation marks. Confidence through restraint.

## Tone
Surgical. Every word earns its place. Status messages read like log lines, not notifications.

## Anti-references
- Military ops center aesthetics (hexagons, radar screens, neon green on black)
- Glassmorphism dashboards
- "Hacker" UI with particle effects and matrix rain
- Template-looking card grids with emoji icons
- Overly animated interfaces with bouncing elements
- Navy/slate color schemes that every "dark dashboard" uses

## References
- Vercel dashboard: extreme whitespace discipline, monospace for data, every element earns its place
- Linear: information density done right, smooth micro-interactions, dark mode that feels intentional
- Stripe dashboard: data-first, clean typography hierarchy, charts that inform not decorate
- Clerk dashboard: modern auth product UX, clear status communication

## Strategic Principles
1. Data over decoration. If an element doesn't convey information, remove it.
2. Monospace for machine-generated data. Always.
3. Color is signal, never decoration. Red means danger. Green means resolved. Indigo means interactive.
4. Motion is functional. Animate only to show state transitions, never for delight.
5. The demo must feel fast. Under 10 seconds from attack to containment, and the UI must communicate that speed.
